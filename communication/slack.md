# AtlasVanguard Slack Operating Rules

This document defines Slack usage within AtlasVanguard.

Slack is for coordination only. All decisions and durable artifacts are recorded in the GitHub governance repository.

---

## 1. Role of Slack

Slack is used for:

- SIG coordination
- RFC discussion
- operational communication
- early research exchange

Slack is not a system of record.

---

## 2. System of Record

The authoritative sources are:

- GitHub governance repository
- RFC system
- SIG charters
- Steering Committee decisions log

All decisions must be recorded in GitHub.

---

## 3. Channel Structure

### Core channels
- `#announcements` (read-only governance updates)
- `#general` (coordination only)

### SIG channels
- `#sig-quant`
- `#sig-risk`
- `#sig-market-structure`
- `#sig-data-infrastructure`
- `#sig-observability-communication`

---

## 4. Decision Rules

- No decisions are made in Slack
- All decisions require RFC or Steering Committee log entry
- Slack discussions are advisory only
- All decisions must reference a GitHub artifact

---

## 5. SIG Responsibilities

SIGs may:

- coordinate execution
- discuss design options
- prepare RFCs

SIGs may not:

- approve cross-SIG changes
- modify governance rules
- define system-wide schemas unilaterally

---

## 6. Communication Standards

- Prefer structured messages
- Link to GitHub artifacts instead of duplicating content
- Separate discussion (Slack) from decision (GitHub)

---

## 7. RFC Workflow

1. Discussion in SIG or RFC channel
2. Draft RFC in GitHub
3. Review by SIGs
4. Steering Committee review if required
5. Final decision recorded in GitHub

---

## 8. Enforcement

Governance rules are enforced by:

- SIG maintainers
- Steering Committee
- RFC review process

---

## 9. Principle

GitHub defines the system. Slack coordinates it.
